# Old Car Bazar — Deployment Guide

Yeh project 4 services par chalti hai:

| Layer    | Service           | Status                                |
| -------- | ----------------- | ------------------------------------- |
| Frontend | Vercel            | Deploy karna hai (Next.js 16)         |
| Backend  | Render.com        | Deploy karna hai (Django + DRF)       |
| Database | Neon Postgres     | Already hosted (`DATABASE_URL`)       |
| Media    | Cloudinary        | Already configured (`CLOUDINARY_*`)   |

---

## 0. Pre-deploy checklist

1. `backend/.env` git mein commit **mat** karein (already gitignored).
2. Production ke liye new `SECRET_KEY` generate karein:
   ```bash
   python -c "import secrets; print(secrets.token_urlsafe(50))"
   ```
3. Neon DB pe `sslmode=require` likha hona chahiye (already hai aapke URL mein).
4. Cloudinary credentials handy rakhein (already `.env` mein).

---

## 1. Backend → Render.com

### a) Code push karein
```bash
git add backend
git commit -m "chore(backend): production deploy config"
git push origin main
```

### b) Render dashboard
1. https://render.com par signup / login.
2. **New + → Blueprint** select karein, repo connect karein.
3. Render `backend/render.yaml` auto-detect karega — `old-car-bazar-api` service create hogi.

### c) Environment variables set karein (Render → Service → Environment)

| Key                        | Value                                                                                  |
| -------------------------- | -------------------------------------------------------------------------------------- |
| `SECRET_KEY`               | (auto-generated by Render, ya manually paste)                                          |
| `DEBUG`                    | `False`                                                                                |
| `ALLOWED_HOSTS`            | `old-car-bazar-api.onrender.com` *(actual hostname use karein)*                        |
| `DATABASE_URL`             | Neon ka full URL                                                                       |
| `CORS_ALLOWED_ORIGINS`     | `https://your-app.vercel.app` *(frontend URL — deploy ke baad fill karenge)*           |
| `CLOUDINARY_CLOUD_NAME`    | aapki cloud name                                                                       |
| `CLOUDINARY_API_KEY`       | aapki key                                                                              |
| `CLOUDINARY_API_SECRET`    | aapki secret                                                                           |
| `JWT_ACCESS_LIFETIME_MIN`  | `60`                                                                                   |
| `JWT_REFRESH_LIFETIME_DAYS`| `30`                                                                                   |
| `OTP_PROVIDER`             | `console`                                                                              |

### d) Deploy
- Render automatically `backend/build.sh` chalayega:
  - `pip install -r requirements.txt`
  - `collectstatic`
  - `migrate`
- Phir Gunicorn server start hoga.
- Health check `/api/schema/` pe configured hai.

### e) Verify
```
GET  https://old-car-bazar-api.onrender.com/api/docs/
GET  https://old-car-bazar-api.onrender.com/api/v1/listings/
```

> **Note:** Free Render service 15 minutes idle ke baad sleep ho jati hai. First request slow (cold start ~30 sec) hoga.

## 2. Frontend → Vercel

### a) Code push karein
```bash
git add .
git commit -m "chore: deploy config"
git push origin main
```

### b) Vercel dashboard
1. https://vercel.com par signup / login (GitHub se).
2. **Add New → Project** → repo import karein.
3. Framework: **Next.js** auto-detect ho jayega.
4. Root Directory: `/` (default).
5. Build/Output settings ko default chhod dein.

### c) Environment variables (Vercel → Project → Settings → Environment Variables)

| Key                   | Value                                                |
| --------------------- | ---------------------------------------------------- |
| `NEXT_PUBLIC_API_URL` | `https://old-car-bazar-api.onrender.com/api/v1`      |

Three environments mein add karein: **Production**, **Preview**, **Development**.

### d) Deploy
- Vercel automatically `npm run build` chalayega.
- Domain milega: `https://your-app.vercel.app`.

### e) Backend ko frontend URL bata do
Render dashboard pe wapas jaayein → `CORS_ALLOWED_ORIGINS` mein Vercel URL add karein:
```
CORS_ALLOWED_ORIGINS=https://your-app.vercel.app
```
Service redeploy ho jayegi automatically.

## 3. Database (Neon) — already done

Neon free tier:
- Auto-pause idle databases (fine for free plan).
- Backups: Neon dashboard se branch-based backups available.

Migrations production pe automatic chalti hain (`build.sh` mein `migrate` hai).

Manually migrate karna ho:
```bash
cd backend
./.venv/Scripts/python.exe manage.py migrate
```

## 6. Common gotchas

| Problem                                | Fix                                                                 |
| -------------------------------------- | ------------------------------------------------------------------- |
| `DisallowedHost` error                 | `ALLOWED_HOSTS` mein hostname add karein                            |
| CORS error from frontend               | `CORS_ALLOWED_ORIGINS` mein exact origin (with protocol) add karein |
| 500 on first request                   | Render cold start hai, 30 sec wait karein                           |
| `static files not found`               | `collectstatic` build mein nahi chala — `build.sh` permissions check |
| `psycopg` install fail                 | Render `requirements.txt` mein `psycopg[binary]` use kar raha hai ✓ |

## 7. Custom domain (optional)

### Frontend
Vercel → Project → Settings → Domains → Add `yourdomain.com`. DNS records Vercel show karega.

### Backend
Render → Service → Settings → Custom Domains → Add `api.yourdomain.com`. CNAME bana lo apne registrar pe.

Phir `ALLOWED_HOSTS` aur `CORS_ALLOWED_ORIGINS` mein new domain bhi add karein.
